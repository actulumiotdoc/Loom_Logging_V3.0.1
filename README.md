 ## Program Loom Logging V3.0.1
**ฐานข้อมูล**
- [x] เปลี่ยนจาก file system เป็น SQL Database
- *การเก็บข้อมูลเป็นไฟล์ทำให้มีเนื้อหาขนาดใหญ่มาก และการเขียนไฟล์แต่ละครั้งต้องทำการเขียนฟังก์ชันการตรวจสอบความถูกต้องของข้อมูล(Transaction)ขึ้นมาเอง
การใช้งาน SQL Database ตัวฐานข้อมูลเองมีการจัดการเรื่องของ Transaction ในตัวอยู่แล้ว จะสามารถตัดปัญหาเรื่องความผิดพลาดในการเขียนข้อมูลได้ และ SQL Database มีขนาดที่เบามาก เมื่อเทียบกับ file system จึงสามารถเก็บข้อมูลย้อนหลังได้เยอะมาก*

**รูปแบบข้อมูล**
- [x] แยก table ข้อมูลระหว่าง Local Server กับ Cloud Server
- *เรามีการใช้งานฐานข้อมูลอยู่ 2 ตัวคือ Local Server (IoT Office) และ Cloud Server (Grafana Dashboard) ดังนั้น ฐานข้อมูลจะแยกเก็บข้อมูลเป็น 2 ตาราง(table) เพื่อให้จัดการเรื่องของการส่งข้อมูลได้ง่าย และแยกระบบอย่างชัดเจน*
  
**การตั้งค่าสถานะล่าสุดของอุปกรณ์(File Config)**
- [x] เปลี่ยนจากไฟล์ตั้งค่า เป็นการ Query จากข้อมูลล่าสุด และนำข้อมูล timestamp มาเป็นตัวกำหนดค่าเริ่มต้น
- *การเขียนไฟล์จดจำสถานะการทำงานล่าสุดของอุปกรณ์Orange Pi (File Config)เพื่อให้ไม่เกิดปัญหาและทำงานอย่างต่อเนื่องของอุปกรณ์ขณะที่อปุกรณ์ถูกปิดและเปิดใหม่ จึงต้องมีการทำไฟล์จดจำสถานะอปุกรณ์ แต่ file config เดิมยังคงใช้ file system (.csv) อยู่ ทำให้เกิดข้อผิดพลาดในการเขียนข้อมูลได้ จะทำให้เกิดปัญหา ไฟล์คอนฟิกได้ จึงเปลี่ยนมาใช้ SQL DataBase แทนเพราะมี่สวนของ Transaction*
  
**การส่งข้อมูล**
- [x] แยกการส่งข้อมูลเป็น 2 ส่วน ตามฐานข้อมูล
- *Local Server ใช้ http node, Cloud Server ใช้ Python Script file*

**เพิ่มหน้าต่างตั้งค่า (Dashboard)**
- [x] ตั้งค่าตัวแปร PLC ผ่าน Dashboard
- *เพิ่มหน้าตั้งค่าขนาดลูกกลิ้งและจำนวนเฟืองเกีบร์*

**logging อุปกรณ์**
- [x] เก็บการทำงานของกล่องทุก 1 วิ
- *เก็บบันทึกการทำงานของอุปกรณ์ตั้งแต่เปิดกล่อง เก็บรวมใน SQL Table ทั้งหมด เก็บย้อนหลัง 7 วันสำหรับ Debug ย้อนหลัง*
